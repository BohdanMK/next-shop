# Гайд по медіафайлах для контент-менеджерів

Цей документ описує які зображення і з якими параметрами потрібно завантажувати через адмінку.

---

## 1. Логотип компанії

| Параметр | Значення |
|---|---|
| Рекомендований розмір | **300×300 px** |
| Мінімальний розмір | 170×170 px |
| Формат | PNG (з прозорим фоном) або JPG |
| Максимальний розмір файлу | 200 КБ |

**Як виглядає:** відображається в хедері та футері сайту у вигляді кола. Зображення автоматично обрізається по круглій масці.

> Логотип повинен бути **квадратним** (однакова ширина і висота), інакше він буде деформований.

## 2. Фото товару (продукту)

| Параметр | Значення |
|---|---|
| Рекомендований розмір | **900×600 px** |
| Мінімальний розмір | 610×406 px |
| Співвідношення сторін | **3:2** (горизонтальне) |
| Формат | JPG або WebP |
| Максимальний розмір файлу | 500 КБ |

**Де відображається:**
- Картка товару в каталозі (305×203 px, на весь контейнер)
- Сторінка окремого товару (велике фото на повну ширину)
- Модальне вікно товару (обрізається по висоті з `object-cover`)
- Мініатюра в кошику (85×56 px, обрізається по `object-cover`)

> Уникайте надмірних відступів навколо страви — займайте якомога більше кадру.

|---|---|

## Загальні рекомендації

### Формати
| Формат | Коли використовувати |
|---|---|
| **JPG** | Фото страв, банери (без прозорого фону) |
| **PNG** | Логотип, іконки інгредієнтів, накладні зображення (потрібен прозорий фон) |
| **WebP** | Найкращий вибір для будь-чого — менша вага при тій самій якості |
| **SVG** | Тільки для простих іконок та іконок категорій (не для фото) |

### Якість та оптимізація
- Зберігайте JPG з якістю **80–85%** — оптимальний баланс якості і розміру
- Не завантажуйте зображення більше **1 МБ** — це сповільнює завантаження сторінки
- Назви файлів пишіть латиницею без пробілів: `chicken-noodles.jpg`, а не `Куряча локшина.jpg`

### Кольоровий простір
- Використовуйте **sRGB** (стандарт для вебу)
